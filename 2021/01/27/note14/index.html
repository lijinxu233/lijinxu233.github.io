1、HashSet、TreeSet、LinkedHashSet

1.1、Set

  • 特点：无序、唯一（不重复）

1.2、HashSet

  • 采用Hashtable哈希表存储结构（神奇的结构）
  • 优点：添加速度快 查询速度快 删除速度快
  • 缺点：无序

1.3、LinkedHashSet

  • 采用哈希表存储结构，同时使用链表维护次序

  • 有序（添加顺序）（索引的顺序）

1.4、TreeSet

  • 采用二叉树（红黑树）的存储结构

  • 优点：有序（按照大小的顺序） 查询速度比List快（按照内容查询）

  • 缺点：查询速度没有HashSet快

时间复杂度

List：O(n) Tree：O（logn） Hashtable： O（1）

3、哈希表的原理

3.1、引入哈希表

  • 在无序的数组中按照内容查找，效率低下，时间复杂度是O（n）。

  • 在有序数组中按照内容来查找，可以使用折半查找，时间复杂度O（logn）。效率很高，比如有10亿个数据在无序数组中查找需要5亿次，在有序数组中用折半查找，只需要30次。

  • 在二叉平衡树中按照内容查找，时间复杂度O（logn）。

  • 在数组中按照索引查找，不进行比较和计数，直接计算得到，效率最高，时间复杂度O（1）。

问题：按照内容来查找，能否也不进行比较，而是通过计算得到地址，实现类似数组按照索引查询的高效率呢？

有！！！哈希表来实现

前面查找方法的共同特点：通过将关键字值与给定值进行比较，来确定位置。效率取决于比较次数。

理想的方法是：不需要比较，根据给定值能直接定位记录的存储信息。

这样，需要在记录的存储位置与该记录的关键字之间建立一种确定的对应关系 ，使每个记录的关键字与一个存储位置相对应。

3.2、哈希表的结构和特点

hashtable:也叫散列表

特点：快 很快啊 非常的快

结构：结构有很多种 最流行、最容易理解：顺序表+链表

主结构：顺序表

每个顺序表的结点再单独引出一个链表

3.3、哈希表是如何添加数据的

  1. 计算哈希码

    x = key.hashCode()

  2. 根据哈希码计算存储位置

    y = x %11；

  3. 存入指定的位置

    情况1：如果是空（一次添加成功）

    直接添加节点

    情况2：如果不是空的（多次添加成功）

    出现了冲突，调用equals（）和对应链表的元素进行比较，比较到最后，结果都是false（没有相同的内容），创建新节点，存储数据，并加入链表末尾。

    情况3：如果不是空的（不添加）

    出现了冲突，调用equals（）和对应链表的元素进行比较，经过一次或者多次比较后，结果是true（有相同的内容），就不再添加，保证唯一性。

    添加快

    唯一

    无序

3.4、哈希表是如何查询数据的

和添加数据的过程是相同的

结论：哈希表查询速度快

3.5、hashCode和equals

  • hashCode（）：计算哈希码，是一个整数，根据哈希码可以计算出数据在哈希表中的存储位置

  • equals（）：添加时出现了冲突，需要通过equals进行比较，判断是否相同；查询时也需要使用equals进行比较，判断是否相同。

3.7、如何减少冲突

3.7.1、哈希表的长度和表中的记录的比例–装填因子

如果Hash表的空间远远大于最后实际存储的个数，则造成了很大的空间浪费，如果选取小了的话，则容易造成冲突。在实际情况中，一般需要根据最终记录存储个数和关键字的分布特点来确定Hash表的大小，还有一种情况是可能事先不知道最终需要存储的记录个数，则需要动态维护Hash表的容量，此时可能需要重新计算Hash地址。

装填因子 = 表中的记录数/哈希表的长度，4/16 = 0.25 8/16 = 0.5

如果装填因子越小，表明表中还有很多的空单元，则添加发生冲突的可能性越小；而装填因子越大，则发生冲突的可能性就越大，在查找时所耗费的时间就越多。有相关文献证明当装填因子在0.5左右时候，Hash性能能够达到最优。

因此，一般情况下，装填因子取经验值0.5。

3.7.2、哈希函数的选择

直接定址法 平方取中法 折叠法 除留取余法(y = x % 11)

3.7.3、处理冲突的方法

链地址法 开放地址法 再散列法 建立一个公共溢出区

4、HashMap、TreeMap、LinkedHashMap

  • Map集合（映射）

    特点：存储的键值对映射关系，根据key可以找到value

  • HashMap

    • 采用Hashtable哈希表存储结构（神奇的结构）
    • 优点：添加速度快 查询速度快 删除速度快
    • 缺点：key无序

  • LinkedHashMap

    • 采用哈希表存储结构，同时使用链表维护次序
    • key有序（添加顺序）

  • TreeMap

    • 采用二叉树（红黑树）的存储结构
    • 优点：key有序 查询速度没有List快（按照内容查询）
    • 缺点：查询速度没有Hashmap快

5、HashMap源码解析

  • 在JDK1.7及其之前，HashMap底层就是一个table数组+链表来实现的哈希表存储结构

  • 链表的每个节点就是一个Entry，其中包括：键key、值value、键的哈希表存储结构、执行下一个节点的引用next四部分

  • 在调用put方法添加键值时。哈希表三步添加数据原理的具体实现；是计算key的哈希码，和value无关。特别注意：

    1. 第一步计算哈希码时，不仅调用了key的hashCode（），还进行了更复杂的处理，目的是尽量保证不同的key尽量得到不同的哈希码。

    2. 第二步根据哈希码计算存储位置时，使用了位运算提高效率。

    3. 第三步添加Entry时添加到链表的第一个位置，而不是链表末尾。

    4. 第三步添加Entry时发现了相同的key已经存在，就使用新的value替代旧的value，并且返回旧的value。

  • 调用get（）方法根据key获取value。

  • 基本特征：二叉树、二叉查找树、二叉平衡树、红黑树

  • 每个结点的结构
